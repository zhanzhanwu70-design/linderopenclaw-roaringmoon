# 在地 AI 輔助開發環境 — 2 個月務實規劃

## 核心原則
- 使用確定的技術（OpenClaw + Ollama/vLLM）
- claw-code 架構當設計參考，不直接依賴
- 每階段結束要有可演示的產出

---

## 硬體對應（根據主管硬體路線圖）

| 階段 | 硬體 | 用途 |
|---|---|---|
| 第 1 階段 | i7-12700TE + 雙 RTX 4000 SFF Ada (20GB VRAM) | 輕量推理 + 基本 coding |
| 第 2 階段 | + RTX 5090 32GB x2 | 中量任務 + 較大模型 |
| 第 3 階段 | + RTX PRO 6000 96GB x2 | 重量推理 + 夜間研究 |

---

## 2 個月總規劃

### 第 1 個月：打下基礎（0-4 週）

#### Week 1：環境架設
- [ ] 在現有硬體上架設 Ollama（最穩定的在地模型管理）
- [ ] 確認第一階段模型可以正常推理
- [ ] 安裝 OpenClaw 並確認基本功能
- [ ] 建立基本網路隔離確認（完全不走外部 API）

#### Week 2：基礎 coding 功能
- [ ] 確認 OpenClaw 可以對本地程式碼工作
- [ ] 串接 Ollama 模型當推理引擎
- [ ] 建立最基本的 tool calling（檔案讀寫、bash）
- [ ] 第一個內部測試案例

#### Week 3：工作流框架
- [ ] 建立「白天模式」：工程師下 prompt → agent 幫寫程式
- [ ] 建立基本檔案隔離（不同專案不能互看）
- [ ] 建立最基本的對話歷史管理
- [ ] 小範圍 pilot（2-3 人實際使用）

#### Week 4：評估與調整
- [ ] 收集 pilot 用戶回饋
- [ ] 修正最緊急的穩定性問題
- [ ] 確認夜間模式技術可行性
- [ ] 建立基本操作手冊

---

### 第 2 個月：功能完整化（5-8 週）

#### Week 5-6：夜間研究模式
- [ ] 實作夜間排程系統
- [ ] 串接文獻/論文查詢（封閉網路內可達的來源）
- [ ] 建立循環測試自動化腳本
- [ ] 與你們產品的程式碼庫串接

#### Week 7：動態路由（如果時間允許）
- [ ] 實作「難度評估層」
- [ ] 根據問題類型自動選模型
- [ ] 建立簡單的任務分流規則

#### Week 8：收尾與展示
- [ ] 建立 POC 展示用 Demo
- [ ] 整理量化數據（錯誤率下降、開發速度提升）
- [ ] 準備給主管/PM 的報告

## 與 claw-code 的關係

| claw-code 的設計 | 在這個計畫中的對應 |
|---|---|
| Tool wiring 系統 | OpenClaw hooks / tools |
| Bootstrap graph | OpenClaw 啟動流程 |
| Task coordination | 自己做簡單的排程層 |
| Prompt routing | 自己實作難度評估 + 模型分流 |
| Rust 效能優化 | 等日後 RTX 5090 / PRO 6000 到位再評估 |

---

## 風險與對策

| 風險 | 機率 | 對策 |
|---|---|---|
| Ollama 在你們環境不穩定 | 低 | 改用 vLLM |
| OpenClaw 與在地模型整合困難 | 中 | 用 OpenAI-compatible API 轉接 |
| claw-code 架構不適用 | 高（已預期） | 完全基於 OpenClaw 自己設計 |
| RTX 5090 延期 | 中 | 先用 RTX 4000 SFF 撐住第一階段 |
| 2 個月時間不足 | 高 | 果斷取捨夜間模式，優先白天 Coding |

- ✅ 程式碼完全不離開內網
- ✅ 起碼 2-3 人實際使用

### 理想成功標準

- ✅ 夜間自動研究模式可以演示

- ✅ 可以對外人展示 POC
