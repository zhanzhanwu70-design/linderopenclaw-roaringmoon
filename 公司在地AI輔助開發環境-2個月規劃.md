# 在地 AI 輔助開發環境 — 2 個月務實規劃（基於 claw-code）

## 核心原則
- 以 claw-code 為核心框架
- 使用 Ollama / vLLM 當在地推理引擎
- claw-code Rust 版本當最終目標
- 每階段要有可演示的產出

---

## 硬體對應（根據主管硬體路線圖）

| 階段 | 硬體 | 用途 |
|---|---|---|
| 第 1 階段 | i7-12700TE + 雙 RTX 4000 SFF Ada (20GB VRAM) | 輕量推理 + 基本 coding |
| 第 2 階段 | + RTX 5090 32GB x2 | 中量任務 + 較大模型 |
| 第 3 階段 | + RTX PRO 6000 96GB x2 | 重量推理 + 夜間研究 |

---

## 2 個月總規劃

### 第 1 個月：打下基礎（0-4 週）

#### Week 1：環境架設 + claw-code 研究
- [ ] 在現有硬體上架設 Ollama（最穩定的在地模型管理）
- [ ] 確認第一階段模型可以正常推理
- [ ] 下載 claw-code 最新版並研究 Rust 分支狀態
- [ ] 建立基本網路隔離確認（完全不對外送資料）

#### Week 2：claw-code 核心功能
- [ ] 確認 claw-code Python 版本可以運作
- [ ] 串接 Ollama 當推理引擎
- [ ] 建立最基本的 tool calling（檔案讀寫、bash）
- [ ] 如果 Rust 版本已穩定，評估直接用 Rust 版本

#### Week 3：基本 CLI 界面
- [ ] 建立基於 claw-code 的基本 CLI 工具
- [ ] 實作最基本的 coding prompt 流程
- [ ] 建立基本檔案隔離（不同專案不能互看）
- [ ] 小範圍 pilot（2-3 人實際使用）

#### Week 4：評估與調整
- [ ] 收集 pilot 用戶回饋
- [ ] 確認 claw-code 架構是否適用你們的產品線
- [ ] 果斷評估「如果 claw-code 不夠用，需要自己加什麼」
- [ ] 建立基本操作手冊

---

### 第 2 個月：功能完整化（5-8 週）

#### Week 5-6：系統整合
- [ ] 跟你們 AMR 產品程式碼庫串接
- [ ] 實作基本的任務分流（依難度選模型）
- [ ] 建立夜間排程系統（如果 claw-code 架構支援）
- [ ] 建立循環測試自動化腳本

#### Week 7：自定義工具層
- [ ] 如果 claw-code 工具層不夠，建立自己的 wrapper
- [ ] 與你們產品的邊緣裝置環境整合
- [ ] 強化安全邊界（完全不對外送資料）

#### Week 8：收尾與展示
- [ ] 建立 POC 展示用 Demo
- [ ] 整理量化數據（錯誤率下降、開發速度提升）
- [ ] 準備給主管/PM 的報告
- [ ] 決定是否要往 Rust 版本移植

## claw-code 元件對應規劃

| claw-code 內建模組 | 你們要對應做什麼 |
|---|---|
| Bootstrap Graph | 評估啟動流程是否適用 |
| Tool Pool | 建立 AMR 產品專屬工具 |
| Query Engine | 串接 Ollama 推理 |
| Command Graph | 建立你們的 command 集合 |
| Hooks | 建立安全審計鉤子 |
| Skills | 建立 AMR 專屬技能 |

---

## 風險與對策

| 風險 | 機率 | 對策 |
|---|---|---|
| claw-code Rust 版本延誤 | 高 | 用 Python 版本先撐住 |
| Python 版本功能不完整 | 高 | 自己補足必要的工具 |
| 與 AMR 產品整合困難 | 中 | 預留 2 週緩衝時間 |
| RTX 5090 延期 | 中 | 先用 RTX 4000 SFF 撐 |
| 2 個月時間不足 | 高 | 果斷取捨夜間模式 |

## 與 OpenClaw 的取決

這次規劃不用 OpenClaw，改用 claw-code。  
如果你們後來發現 claw-code 真的趕不上進度，可以隨時切回評估 OpenClaw。
